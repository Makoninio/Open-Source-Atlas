01 / SCALE

Popularity is only the outer shell.

Stars and forks make open source look clean and legible. The harder truth is underneath: maintenance load, issue backlog, review work, and social coordination.

02 / TIME

Age changes what a repository has to carry.

Older repositories accumulate more than code. They accumulate assumptions, dependencies, users, requests, conventions, and pressure to remain stable.

03 / LABOR

Open source is a system of concentrated effort.

Even when a repository looks communal, contribution patterns usually reveal a smaller set of people doing repeated, high-friction work.

REPOSITORY SNAPSHOT

TIMELINE

Open source history is visible in repository birth dates.

Earlier projects had time to become standards. Later projects often grow in more crowded ecosystems and move faster, but with less institutional history behind them.
